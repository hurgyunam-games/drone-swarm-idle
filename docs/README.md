# 기획 문서 (docs/)

이 폴더는 **Drone Swarm Idle** 게임의 모든 기획 문서를 담는 공간입니다.

바이브코딩 방식으로 개발 시 AI가 상세한 부분을 임의로 결정하거나 누락하는 것을 방지하기 위해,
**개발 시작 전 기획을 충분히 구체화**하는 것을 원칙으로 합니다.

---

## 기획 절차

### 1단계 — 핵심 컨셉 정의
- 게임의 핵심 재미 요소(Core Loop)를 한 문장으로 정의
- 타겟 유저, 플레이 상황, 감성 키워드 설정
- 레퍼런스 게임 분석 (유사 IDLE 게임 벤치마킹)

### 2단계 — 게임 시스템 설계
각 시스템별로 별도 문서를 작성합니다.

| 시스템 | 문서 위치 | 설명 |
|---|---|---|
| 드론 할당 시스템 | `mechanics/drone-assignment.md` | 드론을 업무별로 분배하는 핵심 플레이 |
| 채집 시스템 | `mechanics/gathering.md` | 자원 채집 종류, 속도, 드론 수 영향 |
| 조합 시스템 | `mechanics/crafting.md` | 자원 조합 레시피, 결과물 정의 |
| 업그레이드 시스템 | `mechanics/upgrade.md` | 드론 성능, 슬롯 수 확장 등 |
| 저장 시스템 | `mechanics/save.md` | 로컬 저장 방식 및 데이터 구조 |

### 3단계 — 수치 기획 (밸런싱)
- 각 시스템의 초기 수치, 성장 곡선 설계
- 플레이 타임 대비 진행 속도 조율
- 문서 위치: `balance/`

### 4단계 — UI/UX 설계
- 화면 목록 및 각 화면의 구성 요소 정의
- 화면 전환 흐름도
- 드론 할당 인터페이스 상세 설계 (핵심 화면)
- 문서 위치: `ui-ux/`

### 5단계 — 개발 전 최종 체크리스트
아래 항목이 모두 문서화되어 있는지 확인 후 개발을 시작합니다.

- [ ] 모든 게임 오브젝트(드론, 자원, 조합물 등) 목록과 속성 정의 완료
- [ ] 모든 화면 목록과 각 화면의 기능 정의 완료
- [ ] 각 시스템 간 데이터 흐름 명확히 정의 완료
- [ ] 저장/불러오기 데이터 구조 정의 완료
- [ ] 초기 밸런스 수치 정의 완료
- [ ] 플레이어가 할 수 있는 모든 액션 목록 정의 완료

---

## 문서 작성 규칙

- 모든 문서는 **Markdown(.md)** 형식으로 작성
- 시스템 하나당 문서 하나 (관심사 분리)
- 수치는 구체적인 숫자로 작성 (예: "빠르다" ❌ → "초당 1개 채집" ✅)
- 미확정 사항은 `TODO:` 태그로 명시하여 누락 방지
- AI에게 개발을 요청할 때는 **반드시 해당 기획 문서를 함께 참조**시킬 것

---

## 폴더 구조

```
docs/
├── README.md          ← 이 파일 (기획 절차 가이드)
├── mechanics/         ← 게임 시스템별 상세 기획
├── balance/           ← 수치 기획 및 밸런싱 테이블
└── ui-ux/             ← 화면 설계 및 UX 흐름
```
