# Planning 문서 가이드

`planning/`은 게임의 목적과 방향을 정의하는 영역입니다.
핵심 질문은 "왜 만들고, 무엇을 만들 것인가?"입니다.

---

## 포함할 내용

- 게임 의도(Design Intent)
- 플레이어 경험 목표(Feel/Fun Pillars)
- 코어 루프와 플레이 리듬
- 타겟 유저/플레이 상황
- UX 방향성과 감성 키워드
- 밸런스 목표(정성적)

---

## 작성 원칙

- 기능 구현 방법보다 플레이어 경험과 목적을 먼저 기술
- 수치/코드 구조 같은 구현 세부는 `spec/`로 위임
- 미확정 항목은 `TODO:`로 명시
