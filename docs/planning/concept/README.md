# 컨셉 문서 가이드 (`planning/concept/`)

이 폴더는 **플레이어가 어떤 느낌과 재미를 경험해야 하는지**를 먼저 정의하는 공간입니다.
시스템/수치보다 앞서, "왜 이 게임을 계속 플레이하고 싶어지는가"를 문장으로 명확히 합니다.

---

## 이 폴더에서 먼저 정할 것

- 플레이어가 느껴야 할 핵심 감정(예: 편안함, 몰입감, 성취감)
- 플레이 중 반복적으로 발생해야 할 재미(예: 선택의 의미, 성장 체감, 조합 발견)
- 플레이어가 자주 하게 될 고민의 종류(예: 단기 효율 vs 장기 투자)
- 플레이 리듬과 기대 경험(짧게 확인해도 만족, 오래 보면 더 깊어지는 구조)

---

## 작성 원칙

- 컨셉 단계에서는 정성적 서술을 우선합니다.
- 기능 설명보다 "플레이어 경험" 설명을 먼저 씁니다.
- 의도가 분명해진 뒤에 `docs/spec/`에서 구조/흐름/계약을 구체화합니다.
- 미확정 내용은 `TODO:`로 남겨 누락을 방지합니다.

---

## 문장 예시

- "이 게임은 부담 없이 켜둘 수 있지만, 1~2개의 중요한 선택은 항상 고민하게 만들어 성장의 주도권을 플레이어가 느끼게 한다."
- "플레이어는 매번 작은 성취를 빠르게 확인하고, 가끔 큰 도약을 통해 장기 플레이의 기대감을 유지한다."
