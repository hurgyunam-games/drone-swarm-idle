# 1단계 — 플레이어 경험 중심 컨셉 템플릿

이 문서는 **Drone Swarm Idle**의 방향성을 정리하기 위한 컨셉 문서입니다.
이 단계에서는 구체적인 수치보다 **플레이어가 느낄 감정과 재미의 구조**를 우선 정의합니다.

---

## 1) 게임 의도 (Design Intent)

- 이 게임을 만들고 싶은 이유:
  - 누워서 한 손으로도 깊이 있는 자동화/운영 게임을 즐기고 싶다.

- 플레이어에게 제공하고 싶은 핵심 경험:
  - 불완전한 현실 대신, 내가 설계한 규칙과 정책으로 시스템이 유기적으로 굴러가는 세상을 만들며 성취감과 통제감을 느끼게 하고 싶다.

- 이 게임의 한 줄 방향성:
  - 모바일 친화 조작으로, 림월드/팩토리오급 전략 깊이를 담은 군집 자동화 운영 경험을 제공한다.

---

## 2) 플레이어 경험 목표 (Feel/Fun Pillars)

- 플레이어가 반드시 느껴야 할 감정 3가지:
  1. 규칙을 짜는 창조감
  2. 의도대로 작동할 때의 설계자 만족감
  3. 시행착오를 줄여갈 때의 성장감

- 이 감정이 재미로 연결되는 이유:
  - 직접 조작보다 규칙 설계의 결과가 누적되어, 플레이어가 "세계의 운영자"라는 정체성을 강하게 느끼기 때문이다.

- "이 게임만의 재미" 한 문장:
  - 손이 아니라 규칙으로 세계를 움직이는 정책 설계형 방치 전략의 재미.

---

## 3) Core Loop 한 문장 정의

> 예시 형식: "플레이어는 [행동]을 통해 [자원/성과]를 얻고, 이를 [강화/확장]에 사용해 더 큰 성장을 만든다."

- Core Loop 문장:
  - 플레이어는 정책과 우선순위를 설계해 드론 군집의 작업 흐름을 운영하고, 생산/물류 성과를 시설 업그레이드와 군집 확장에 재투자해 더 큰 처리량과 새로운 운영 문제를 만든다.

- 루프 단계(3~5단계 권장):
  1. 관찰: 대시보드에서 병목(충전 대기, 운반 지연, 처리 정체)과 목표 미달 구간을 확인한다.
  2. 설계: 작업 우선순위, 허용 구역, 역할 비율(수집/운반/제작/정비) 같은 군집 정책을 조정한다.
  3. 실행: 드론 군집이 정책에 따라 자동으로 작업을 수행하며 자원과 생산량을 누적한다.
  4. 강화: 확보한 성과로 설비/충전 인프라/알고리즘을 업그레이드하고 군집 규모를 확장한다.
  5. 확장: 처리량 증가로 신규 구역과 고급 체인이 열리며 더 복잡한 병목이 발생해 다음 최적화 루프로 이어진다.

- 루프가 재미있는 이유(핵심 보상):
  - 작은 정책 변경이 즉시 처리량/안정성 개선으로 보이고, 단기 최적화 쾌감과 장기 확장 성취가 동시에 누적된다.

## 7) 레퍼런스 게임 분석 (벤치마킹)

| 게임명 | 참고할 의도/경험 요소 | 우리 게임에 적용할 점 | 피할 점 |
|---|---|---|---|
| RimWorld | 제한된 자원/인력에서 발생하는 우선순위 운영의 긴장감 | 드론 역할 우선순위(수집/운반/제작/정비)와 작업 큐 충돌을 통해 "운영 판단의 재미" 강화 | 과도한 이벤트 난이도와 페널티로 피로감 유발 |
| Factorio | 생산 라인이 맞물리며 병목을 해소하는 자동화 최적화 쾌감 | 드론 물류 네트워크를 시각화하고 병목 구간(충전, 운반, 처리)을 명확히 보여 개선 동기 제공 | 초기 진입장벽이 높은 복잡한 레시피 구조 |
| Dyson Sphere Program | 거대 목표를 향해 시스템을 확장할 때 느끼는 스케일 성장감 | "드론 군집 확장 -> 처리량 폭증 -> 다음 구역 해금"의 장기 성장 목표 설계 | 중후반 반복 작업 과다로 인한 루틴 피로 |
| Oxygen Not Included | 간접 제어와 환경 제약 속에서 문제를 해결하는 퍼즐형 운영 재미 | 드론에게 직접 명령 대신 정책(우선순위/허용 구역/작업 조건) 기반 제어 도입 | 시스템 상호작용 과복잡화로 인한 학습 부담 |
| Mindustry | 짧은 루프 안에서 물류 구축과 방어 판단이 동시에 요구되는 몰입감 | "짧은 체크 -> 즉시 효율 개선"이 가능한 가시적 피드백 UI/알림 설계 | 전투 압박이 과도해 방치형의 편안함을 해치는 구성 |

### 레퍼런스별 핵심 재미 간단 분석

- **RimWorld**: "완벽한 계획"보다 "제한 속 우선순위 결정"이 핵심 재미다. 예측 불가능한 변수 속에서 어떤 작업을 먼저 살릴지 선택하게 만들면, 플레이어는 시스템의 주인으로 느낀다.
- **Factorio**: 재미의 본질은 생산량 숫자 자체가 아니라 병목을 발견하고 라인을 재배치해 흐름이 살아나는 순간의 성취감이다. 즉, 문제 인지 -> 구조 개선 -> 즉시 성과 확인의 반복이 강력하다.
- **Dyson Sphere Program**: 장기 목표가 명확할수록 중간 자동화 단계도 의미를 얻는다. "지금의 작은 확장"이 "거대한 시스템"으로 연결된다는 감각이 플레이 지속 동기를 만든다.
- **Oxygen Not Included**: 플레이어가 직접 모든 행동을 하지 않고 규칙을 설계해 문제를 풀게 하는 간접 통제형 재미가 강하다. 이는 드론 운영 게임에서 정책 설계의 재미로 이식하기 좋다.
- **Mindustry**: 정보 피드백이 빠르고 명확해서 의사결정 속도가 올라간다. 방치형에서도 짧은 확인 시간 안에 "어디를 고치면 좋아지는지"가 보여야 핵심 재미가 살아난다.

- 공통 패턴 요약:
  - 병목을 찾고 해소하는 과정 자체가 핵심 재미이며, 이를 위해 시스템 상태가 명확히 "보여야" 한다.
  - 단기 루프(즉시 개선 체감)와 장기 루프(확장 목표 달성)가 동시에 작동할 때 플레이 유지력이 높아진다.
  - 플레이어는 미시 조작보다 규칙/우선순위/배치 설계 같은 거시적 통제에서 더 큰 만족을 얻는다.

- 차별화 포인트(우리만의 강점):
  - 드론 "개체"가 아닌 드론 "군집"을 다루는 감각(역할 비율, 군집 정책, 집단 행동 최적화)을 전면에 둔다.
  - 방치형 리듬에 맞춰 짧은 접속에서도 전략적 결정을 내릴 수 있는 경량 운영 경험을 지향한다.
  - 물류/생산 최적화에 감성적 몰입(군집이 살아 움직이는 시각 피드백)을 결합해 "보는 재미"를 강화한다.
