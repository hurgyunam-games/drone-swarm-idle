# RimWorld 역기획 노트

이 디렉터리는 RimWorld를 분석해 내 게임 기획 역량으로 전환하기 위한 작업 공간이다.

## 목적
- RimWorld의 핵심 재미를 시스템 관점에서 분해한다.
- "왜 재미있는가"를 재사용 가능한 기획 원칙으로 정리한다.
- 분석 결과를 내 프로젝트의 실행 가능한 백로그로 변환한다.

## 폴더 역할 지도 (01-08)
- `01_core_loop/`: 게임의 가장 큰 반복 구조를 정의한다. (관찰 -> 판단 -> 지시 -> 사건 -> 복구)
- `02_player-experience/`: 플레이 감정 곡선, 마찰, 억울함/성취 지점을 정리한다.
- `03_systems/`: 핵심 시스템을 하나씩 분해한다. (규칙, 입력, 출력, 결정, 실패/회복)
- `04_content-structure/`: 바이옴, 세력, 이벤트, 아이템처럼 콘텐츠 묶음을 구조화한다.
- `05_balance/`: 리스크-보상, 난이도 스케일링, 초중후반 체감 변화를 다룬다.
- `06_meta/`: DLC/모드가 기본 구조에 주는 변형을 비교한다.
- `07_benchmarks/`: 장르 내 타 게임과 비교해 RimWorld의 차별점을 뽑는다.
- `08_outputs/`: 분석 결과를 내 프로젝트용 원칙/실행 태스크로 변환한다.

## 번호 체계 설명
- 번호는 "중요도"가 아니라 "분석 목차(레이어)"를 뜻한다.
- `01 -> 02 -> 03`은 거시 구조(루프/경험)에서 미시 구조(시스템)로 내려가는 분류 순서다.
- 실제 작업 시작점은 학습 난이도 때문에 `03_systems/`가 더 쉽다.
- 즉, 폴더 번호와 실제 작성 순서는 다를 수 있다.

## 실사용 순서 vs 폴더 번호
- 실사용(초반): `logs/` -> `03_systems/` -> `08_outputs/`
- 종합(중반 이후): `01_core_loop/`, `02_player-experience/`, `05_balance/`
- 번호는 문서 정리용 인덱스, 실사용 순서는 학습 효율 기준으로 운영한다.

## 운영 원칙
- 플레이 감상문보다 "규칙-선택-결과" 구조를 우선 기록한다.
- 매 문서 끝에는 반드시 `내 게임 적용 아이디어`를 작성한다.
- 추측은 추측으로 표시하고, 관찰과 해석을 분리한다.

## 문서 작성 스타일 룰
- 처음에는 자유 기록을 허용하고, 이후 `선택-대가-감정` 3줄 결론으로 가공한다.
- 수치가 없으면 빈칸으로 두지 말고 빈도/강도 표현(자주, 크게, 즉시 등)을 사용한다.
- 한 문서의 목표는 "완벽 정리"가 아니라 "다음 실험 질문 1개 만들기"다.
- 시스템 문서는 기능 설명보다 "어떤 결정을 강제하는가"를 우선 작성한다.

## 기본 진행 루틴
1. 플레이 세션 기록 (`logs/`)
2. 시스템 단위 분석 (`03_systems/` 또는 템플릿 활용)
3. 적용 항목 추출 (`08_outputs/actionable-backlog.md`)

## 권장 시작 순서 (초반 2주)
1. `00_scope-and-rules.md` 작성
2. `logs/`에 자유 기록 작성 + `선택-대가-감정` 3줄 결론 2~4개 작성
3. `03_systems/`에서 가장 강하게 느낀 시스템 1개만 분석
4. `08_outputs/actionable-backlog.md`에 실험 태스크 1~2개 등록

## 왜 초반에 `03_systems/`부터 시작하나?
- 초보 단계에서는 범위를 넓게 잡으면 금방 피로해진다.
- `03_systems/`는 "한 시스템씩" 작게 쪼개 분석할 수 있어 진입 장벽이 낮다.
- 기록을 즉시 분석으로 연결하기 쉬워 학습 속도가 빠르다.
- `01`, `02`, `05`는 시스템 문서가 2~3개 쌓인 뒤 종합할 때 정확도가 높아진다.
