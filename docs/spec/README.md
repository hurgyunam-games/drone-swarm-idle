# Spec 문서 가이드

`spec/`은 구현을 위한 기술 명세를 작성하는 영역입니다.
핵심 질문은 "어떻게 만들 것인가?"입니다.

---

## 포함할 내용

- 요구사항/범위 및 완료 조건
- 클래스 다이어그램(구조)
- 시퀀스 다이어그램(흐름)
- 인터페이스 계약(입력/출력/이벤트/에러)
- 상태 전이/비즈니스 규칙
- 테스트 시나리오/인수 기준

---

## 작성 원칙

- 코드 구현 전에 문서만 보고 작업 가능해야 함
- 다이어그램과 텍스트 명세(규칙/계약/테스트)를 함께 유지
- 변경 시 관련 문서를 같은 작업 단위로 동기화
